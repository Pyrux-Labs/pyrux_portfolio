## Directory Purposes

**`app/`:**
- Purpose: Next.js App Router routes and global layout
- Contains: Server Components for pages (metadata), Client Components for interactivity
- Key files: `page.tsx` (Server), `*PageClient.tsx` (Client), `layout.tsx` (Root), `globals.css` (Design system)

**`components/`:**
- Purpose: Reusable React components organized by type
- Contains: Sections (full-width content blocks), Cards (repeatable UI elements), Modals (overlay dialogs), UI primitives (helpers)
- Key files: Each subdirectory has focused responsibility

**`components/sections/`:**
- Purpose: Large, self-contained content sections
- Contains: Hero, Projects, Team, Services, FAQ, Contact, etc.—each renders a major homepage section
- Pattern: Most are Client Components (`"use client"`) to manage state and animations
- Example: `OurProjects.tsx` manages modal state and carousel interactions

**`components/cards/`:**
- Purpose: Repeating card UI elements
- Contains: ProjectCard, CompanyCard, PackageCard, MaintenanceCard
- Pattern: Receive data object + optional handlers as props; Framer Motion for animations
- Placement: Used in carousels, grids, and lists throughout the app

**`components/modals/`:**
- Purpose: Detail/richview overlays
- Contains: ProjectModal, CompanyModal, CreatorModal
- Pattern: Wrap the base `Modal` component; receive data + onClose handler
- Trigger: Click on a card → parent component sets state → modal appears

**`components/ui/`:**
- Purpose: Low-level UI primitives and utilities
- Contains: Generic `Modal`, `Section`, `Badge`, `StarBackground`, `ThemeToggle`
- Pattern: Accept minimal props, maximum reusability, no domain-specific logic

**`data/`:**
- Purpose: Static content stored as TypeScript arrays
- Contains: All business content (projects, companies, FAQs, pricing) and helper functions
- Pattern: Exported arrays + utility functions (e.g., `getCreatorById()`, `getFeaturedProjects()`)
- Benefit: Single edit point; no database; fully type-safe

**`types/`:**
- Purpose: Shared type definitions
- Contains: Domain models (Project, Company, Creator, Technology, etc.)
- Key file: `types/index.ts` (centralized, named exports)
- Pattern: Used throughout components and data files; never redefined locally

**`lib/`:**
- Purpose: Utility functions and helpers
- Contains: `cn()` for Tailwind class merging, potentially string formatters, date utilities
- Pattern: Pure functions, no React dependencies, reusable across components

**`hooks/`:**
- Purpose: Custom React hooks
- Contains: `useCopyToClipboard` for clipboard operations, potentially theme management
- Pattern: Encapsulate stateful logic; exported as named functions; used in Client Components

**`public/`:**
- Purpose: Static assets served at root level
- Contains: SVG logos, PNG images (Open Graph, project screenshots, company logos)
- Organization: Images grouped by entity type/ID for clarity
- Build: Assets are copied as-is to `out/` during static export

**`out/`:**
- Purpose: Final build artifact (static HTML + assets)
- Generated: `npm run build` → outputs to `out/`
- Deployment: `out/` is pushed to GitHub Pages via GitHub Actions
- Not committed: Usually in `.gitignore` (temporary)

## Naming Conventions

**Files:**
- **Pages:** `page.tsx` (Next.js standard)
- **Client-only pages:** `PageClient.tsx` or `RoutePageClient.tsx` (e.g., `ProjectsPageClient.tsx`)
- **Components:** PascalCase (`Hero.tsx`, `ProjectCard.tsx`, `FAQAccordion.tsx`)
- **Utilities/Hooks:** camelCase (`useCopyToClipboard.ts`, `utils.ts`)
- **Data files:** camelCase plural (`projects.ts`, `companies.ts`)
- **Types:** camelCase plural in `types/index.ts`

**Directories:**
- **App routes:** lowercase, match URL structure (`app/pricing/`, `app/creator/[id]/`)
- **Component categories:** lowercase plural (`sections/`, `cards/`, `modals/`, `ui/`)
- **Data:** singular `data/`
- **Utilities:** singular `lib/`, `hooks/`

**Component Props:**
- **Interfaces:** PascalCase with `Props` suffix (e.g., `ProjectCardProps`)
- **Props are typed:** Never use `any`; always define interfaces
- **Destructure in parameters:** `function Component({ prop1, prop2 }: ComponentProps)`

**CSS Classes:**
- **Utility-first:** Tailwind CSS only (no inline styles, no CSS modules)
- **Responsive prefixes:** `sm:`, `md:`, `lg:` for mobile-first breakpoints
- **Theme colors:** Use CSS variables (e.g., `text-primary`, `bg-coral`)
- **Semantic naming:** Class names describe function, not appearance (e.g., `hover:border-coral` not `hover:red`)

**Variables & Constants:**
- **Tailwind colors:** Referenced via CSS variables in `:root` (e.g., `--coral-bright`, `--text-primary`)
- **Animation durations:** Hard-coded in Framer Motion variants (e.g., `transition: { duration: 0.6 }`)
- **Environment:** No env vars needed (fully static)

## Where to Add New Code

**New Feature/Section:**
- **Primary code:** `components/sections/NewSectionName.tsx`
- **Data source:** `data/newEntities.ts` (if needed)
- **Add to homepage:** Import in `app/page.tsx` and place in layout
- **New type:** Add to `types/index.ts`

**New Page (route):**
- **Create:** `app/newroute/page.tsx` (Server Component for metadata)
- **Create:** `app/newroute/NewRoutePageClient.tsx` (Client Component for UI)
- **In page.tsx:** Import PageClient and render it
- **Metadata:** Export `metadata` object and `generateMetadata()` if dynamic

**New Card Component:**
- **Create:** `components/cards/NewCard.tsx`
- **Pattern:** Accept typed props (`NewCardProps`), use Framer Motion for animations
- **Use in:** Sections or grids that import and map over arrays

**New Modal:**
- **Create:** `components/modals/NewModal.tsx`
- **Pattern:** Wrap `<Modal>` component, receive data + `onClose` handler
- **Trigger:** Parent component manages state (`useState`) and passes `isOpen` + `onClose`

**New Utility:**
- **Helper function:** `lib/utils.ts` (exported named function)
- **Custom hook:** `hooks/useNewHook.ts` (exported named function, use `use` prefix)
- **Pattern:** Keep pure and reusable; no side effects unless necessary

**New Global Style:**
- **CSS variables:** Add to `:root` in `app/globals.css`
- **Tailwind animation:** Add to `@layer utilities` section
- **Light mode override:** Add under `html[data-theme="light"]` if needed

## Special Directories

**`public/`:**
- Purpose: Serve static assets at root URL
- Generated: No; manually managed
- Committed: Yes; images and logos are part of source
- Access in code: `<Image src="/projects/..." />` or `<img src="/logo.svg" />`

**`out/`:**
- Purpose: Static export build artifact
- Generated: Yes; by `npm run build`
- Committed: No (usually in `.gitignore`)
- Deployment: Contents pushed to GitHub Pages via GitHub Actions

**`.next/`:**
- Purpose: Next.js internal build cache
- Generated: Yes; by `npm run dev` or `npm run build`
- Committed: No (in `.gitignore`)
- Purpose: Speed up development builds

## Design System Reference

**Colors (CSS Variables in `app/globals.css`):**
- `--coral-bright` (#ff7b3d, dark mode) — Primary accent color
- `--cyan-bright` (#d4a574, dark mode) — Secondary accent (gold/tan)
- `--text-primary` (#fef7f0) — Primary text
- `--text-secondary` (#c9b5a0) — Muted text
- `--bg-deep` (#0a0604) — Darkest background
- `--bg-surface` (#120d0a) — Default background
- `--bg-elevated` (#1a130f) — Elevated/card background

**Animations:**
- **Fade-in on scroll:** `whileInView={{ opacity: 1 }}` with variants
- **Stagger children:** `transition: { staggerChildren: 0.06 }`
- **Spring animations:** `type: "spring"`, `damping: 25`, `stiffness: 300`
- **Hover lift:** `whileHover={{ y: -4 }}` or `whileHover={{ y: -2 }}`

**Typography:**
- **Font:** Manrope (Google Fonts, loaded in `layout.tsx`)
- **Display heading:** `font-display text-[clamp(2.8rem,8vw,6rem)] font-bold`
- **Section title:** `font-display text-[1.4rem] font-semibold`
- **Body:** `text-[1rem] leading-[1.7]`
