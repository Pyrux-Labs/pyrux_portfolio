   - Import `esMessages` from `@/messages/es.json` and `enMessages` from `@/messages/en.json`

   - Create and export `LocaleContext` with type `{ locale: Locale; setLocale: (l: Locale) => void }`
   - Export `useLocale` hook that reads from `LocaleContext`
   - Define `messagesMap: Record<Locale, Record<string, unknown>>` mapping `es` to `esMessages` and `en` to `enMessages`
   - Export `LocaleProvider` component that:
     a. Uses `useState<Locale>("es")` for locale state
     b. In a `useEffect` on mount (empty deps): reads `localStorage.getItem("pyrux-locale")`, if valid ("es"|"en") uses it; otherwise detects via `navigator.language.startsWith("es") ? "es" : "en"`, stores result in localStorage
     c. Provides a `setLocale` function that calls `setLocaleState`, writes to `localStorage.setItem("pyrux-locale", newLocale)`, and sets `document.documentElement.lang = newLocale`
     d. Wraps children in `<LocaleContext.Provider>` then `<NextIntlClientProvider locale={locale} messages={messagesMap[locale]}>`
