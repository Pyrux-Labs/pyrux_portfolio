**BUG-01 — Modal backdrop flicker (Modal.tsx):**

The flicker happens because the overlay and modal content have separate animation lifecycles. The overlay uses `overlayVariants` on the container, while the modal content has its own exit animation (0.2s). When the modal closes, the overlay disappears instantly (AnimatePresence removes it) while the content is still animating out.

Fix approach: Add an exit variant to the overlay that matches the modal content exit duration. Ensure the parent container's `exit` matches too. Specifically:

1. Add exit to overlayVariants: `exit: { opacity: 0, transition: { duration: 0.2 } }`
2. On the parent `motion.div` (the fixed container), add `exit="hidden"` -- this is already present.
3. The key issue: the backdrop `motion.div` (the `bg-black/60` element) has no exit animation of its own. When the parent exits, the backdrop disappears abruptly because it inherits the parent's exit. Add explicit exit to the backdrop:
   - On the backdrop div, add `initial={{ opacity: 0 }}`, `animate={{ opacity: 1 }}`, `exit={{ opacity: 0 }}` with a `transition={{ duration: 0.2 }}`.
   - This ensures the backdrop fades out smoothly instead of cutting.

4. Additionally, wrap the entire `AnimatePresence` with `mode="wait"` to prevent enter/exit overlaps:
   Change `<AnimatePresence>` to `<AnimatePresence mode="wait">`.
